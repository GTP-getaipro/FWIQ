# üöÄ Unified Email Provisioning: Gmail + Outlook Support

## üéØ Overview

This is a **production-ready TypeScript implementation** that handles **both Gmail and Microsoft 365 (Outlook)** OAuth flows in one unified backend function. It automatically provisions n8n workflows for businesses regardless of their email provider.

## üèÜ What This Implementation Provides

‚úÖ **Dual Provider Support** - Gmail and Outlook (Microsoft 365) OAuth flows
‚úÖ **Unified API Endpoint** - Single endpoint handles both email providers
‚úÖ **Provider Detection** - Automatic detection of email provider from OAuth data
‚úÖ **Credential Management** - Creates appropriate n8n credentials for each provider
‚úÖ **Label/Folder Creation** - Gmail labels and Outlook folders via respective APIs
‚úÖ **Workflow Configuration** - Configures nodes based on email provider
‚úÖ **Error Handling & Rollback** - Complete rollback system for failed operations
‚úÖ **Comprehensive Testing** - Full test suite covering both providers

## üß© Architecture Overview

```
[Frontend OAuth] 
   ‚Üì Google OAuth tokens OR Microsoft OAuth tokens
[Backend Unified API]
   ‚Üí Detects email provider (gmail/outlook)
   ‚Üí Creates appropriate n8n credential
   ‚Üí Loads industry configuration
   ‚Üí Clones base workflow template
   ‚Üí Configures workflow nodes for provider
   ‚Üí Creates Gmail labels OR Outlook folders
   ‚Üí Activates workflow
   ‚Üí Stores integration data
[n8n Runtime]
   ‚Üí Gmail Trigger OR Outlook Trigger polls business inbox
   ‚Üí AI Classifier processes emails
   ‚Üí AI Draft Generator creates responses
   ‚Üí Gmail nodes OR Outlook nodes send/draft replies
```

## üîß Implementation Details

### **1. Main Function: `provisionN8nWorkflowForBusiness`**

**Purpose:** Complete workflow provisioning for Gmail or Outlook
**Input:** Business data + OAuth tokens (provider-agnostic)
**Output:** Success/failure with workflow and credential IDs

```typescript
const result = await provisionN8nWorkflowForBusiness(businessData, oauthData);

if (result.success) {
  console.log(`Workflow ID: ${result.workflowId}`);
  console.log(`Credential ID: ${result.credentialId}`);
  console.log(`Email Provider: ${result.emailProvider}`);
} else {
  console.error(`Error: ${result.error}`);
}
```

**Key Features:**
- ‚úÖ **8-Step Process** - Complete workflow provisioning for both providers
- ‚úÖ **Provider Detection** - Automatic detection from business data
- ‚úÖ **Credential Creation** - Gmail or Outlook credentials based on provider
- ‚úÖ **Label/Folder Management** - Gmail labels or Outlook folders
- ‚úÖ **Node Configuration** - Provider-specific node configuration
- ‚úÖ **Rollback System** - Automatic cleanup on failure

### **2. API Endpoint: `handleUnifiedOAuthCallback`**

**Purpose:** Unified Express.js API endpoint for both Gmail and Outlook OAuth callbacks
**Route:** `POST /api/onboarding/email/oauth`
**Input:** Business ID, business data (with emailProvider), OAuth data
**Output:** JSON response with success/error status

